# Phase 0.5 実装計画

作成日: 2026-06-03

## 結論

現MVPは、地図で市場を見るだけのツールから、採用担当者が経営に説明できる採用競争力ダッシュボードへ進める。

最優先は、旧 `重点調査候補 proxy` を1つの総合判断に使わず、以下のスコアへ分解すること。

- 市場魅力度
- 採用難易度
- 求人条件競争力
- 受入/定着リスク
- データ信頼度

## 実装済みのPhase 0.5要素

- エリアタブに分解スコアカードを追加
- `総合探索優先度 proxy` を補助値として扱うよう名称変更
- 市区町村別の給与 P50/P75/P90 を表示
- 競合求人TOP10セクションを追加
- 条件タブに改善メモを追加
- 経営提案用CSV / Markdown出力を追加
- 市区町村CSVに `score_version` と分解スコア列を追加
- 根拠タブにスコア辞書を追加

## スコア定義

### 市場魅力度

その地域で訪問看護事業を伸ばす価値。需要、供給不足、紹介元接点から算出する。

### 採用難易度

看護師を採りにくい強さ。競合密度、供給不足、給与圧、求人件数proxyから算出する。

### 求人条件競争力

条件タブに入力した募集条件が、地域求人と比べて見劣りしにくいか。採用確率ではない。

### 受入/定着リスク

管理者負荷、同行訪問、教育体制、オンコール移行、訪問件数上限など。現MVPでは未接続。

### データ信頼度

求人件数と突合率から算出。他スコアには混ぜず、判断の信頼度として別表示する。

## 次のPhase 1

次は競合求人ベンチマークの実用化を進める。

1. 求人CSV標準スキーマを強化
2. 競合求人TOP10を給与以外でも比較
3. 不足項目アラートを追加
4. 条件改善提案を候補者心理に寄せる
5. 経営提案Markdownを会議資料として使える粒度へ整える

## 注意

採用確率、採用成功率、勝率、獲得率のような表現は使わない。応募・面接・内定・承諾・入職・定着の実績データが入ってから、予測モデルとして別途扱う。
