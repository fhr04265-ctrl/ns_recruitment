# 実求人パイロット投入計画 2026-06-04

## 結論

次のマイルストーンは、首都圏の3〜5市区町村で50〜200件の公開求人を小規模投入し、網羅性ではなく「分析に耐えるか」を検証すること。東京23区だけに寄せず、都市度・訪問看護密度・交通条件・広域訪問の違いが出るように選ぶ。

## 対象候補

- 世田谷区: 訪問看護事業所が多く、競合比較の検証に向く。
- 八王子市または町田市: 面積が広く、通勤・訪問効率、車移動、都県境の論点が出やすい。
- 川崎市または横浜市の1区: 東京隣接エリアとの給与・交通条件差を見やすい。
- さいたま市または川口市: 住宅地・都心通勤圏の条件比較に向く。
- 船橋市または市川市: 千葉側の給与・オンコール・移動条件差を検証しやすい。

## 収集する求人

- 訪問看護師、正社員、日勤、月給レンジありを優先。
- 病院、診療所、施設系看護師も比較母集団として一部含める。
- 求人媒体は公開ページのみ。出典URL、取得日、取得方法、利用区分、レビュー状態を必ず残す。
- 初回は媒体網羅より、給与・休日・オンコール・直行直帰・教育・訪問件数上限の取得率を優先する。

## 必須KPI

- 投入求人件数: 50件以上
- 分析対象求人: 30件以上
- 施設突合率: 60%以上、または接続20件以上
- 平均品質スコア: 65以上
- 要レビュー求人: 40%未満
- 出典URL入力率: 95%以上
- 取得日入力率: 100%
- 給与レンジ入力率: 80%以上
- 年間休日、オンコール、教育、訪問件数上限の主要項目欠損率を確認

## 成功条件

1. `real_job_pilot_template.csv` と同じ60列で投入できる。
2. HTML上で求人タブの品質レビューが表示される。
3. `job_salary_quality_review_mvp.csv` が出力できる。
4. 未突合・候補複数求人を人間が補正できる。
5. 条件タブの競合TOP10と経営提案Markdownが、実求人でも破綻しない。

## 禁止・注意

- 採用確率、獲得率、勝率、何％取れる、という表現は使わない。
- 求人票の未記載は「制度なし」ではなく「求人票上確認できない」と扱う。
- 応募、面接、内定、入職、定着の実績データがない段階では、出力は求人条件競争力と改善優先度に限定する。
