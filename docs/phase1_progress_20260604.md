## 実装内容

### 1. 勝ち/負け/不明ラベル

条件タブの `競合TOP10との差分` で、入力した自社/想定条件と競合求人を比較し、以下のように表示する。

- `勝ち`: 自社/想定条件が競合より有利
- `負け`: 自社/想定条件が競合より不利
- `不明`: 競合求人票に記載がなく比較不能、または同等

比較対象:

- 月給
- 年間休日
- 土日祝休み
- オンコール
- 直行直帰
- 未経験可
- 訪問件数上限

### 2. 欠損値の扱い

空欄や未記載を `0円`、`休日0日`、`制度なし` として扱わない。

求人票上確認できない項目は、`求人票不足項目` として別枠で表示する。

### 3. 求人票不足項目アラート

以下に不足項目アラートを追加した。

- エリアタブ: 選択市区町村の競合求人TOP10に対する不足項目
- 求人タブ: 接続済み求人全体の不足項目
- 条件タブ: 入力条件との競合差分表の上部

注記:

> 不足項目は求人票上確認できない項目です。実際に制度がないという意味ではありません。

### 4. 経営提案出力の監査列

経営提案CSVに以下を追加した。

- `missing_fields_summary`
- `audit_job_ids`
- `audit_gap_labels`
- `notice_type`

これにより、後から「なぜその改善提案になったか」を追いやすくした。
