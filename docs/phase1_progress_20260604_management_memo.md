### 2. 結論文の追加

`managementDecisionLine()` を追加し、以下のような判断文を生成する。

- 求人CSVの接続求人が少ない場合は、条件変更判断を保留
- 市場魅力度が高く採用難度も高い場合は、条件改善を前提に採用強化候補として検討
- 条件競争力が低い場合は、給与だけでなく休日・オンコール・教育訴求を優先
- 条件競争力が高い場合は、求人票不足項目の補完と媒体露出を優先

### 3. 追加コスト目安の注記強化

`costNote()` を追加し、月給平均との差分に基づく追加コスト目安について、以下を明記した。

- 月給下限と地域中央値の差分に基づく概算
- 賞与、法定福利費、採用費、教育工数は含まない
- 現条件を下げる提案ではない

### 4. Markdown表の整形

`markdownTable()` を追加し、判断サマリーと給与条件を表形式で出力するようにした。
